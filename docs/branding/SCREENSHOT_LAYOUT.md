# README Screenshot Layout

## 目标
让 BilBiliChat 的 GitHub 首页在没有太多文字说明时，也能通过截图立刻传达：

- 这是一个真正的桌面客户端
- 它有专业主控制台，而不是简单弹幕工具
- 它支持 Overlay、OpenLive、插件与多窗口工作流

## 推荐顺序
### 1. Hero Screenshot
- 文件：`docs/screenshots/main-console.png`
- 内容：主控制台全景
- 要点：Sidebar、房间列表、状态卡片、主舞台感 UI

### 2. Overlay Screenshot
- 文件：`docs/screenshots/overlay-window.png`
- 内容：透明弹幕输出
- 要点：弹幕、礼物、SC、描边、阴影、状态栏

### 3. Overlay Studio Screenshot
- 文件：`docs/screenshots/overlay-studio.png`
- 内容：模板预设 + 实时舞台预览
- 要点：拖拽手柄、样式控制、礼物与 SC 预览

### 4. Settings Screenshot
- 文件：`docs/screenshots/settings-system.png`
- 内容：OpenLive、更新渠道、配置同步、Overlay 设置
- 要点：产品化设置体系完整度

### 5. Multi Window Screenshot
- 文件：`docs/screenshots/multi-window.png`
- 内容：Main、Danmu、Settings、Login、Overlay Studio 并列
- 要点：多窗口桌面应用质感

## 排版建议
### 第一屏
- 单张宽图：主控制台

### 第二屏
- 两列布局：
  - 左：Overlay Window
  - 右：Overlay Studio

### 第三屏
- 两列布局：
  - 左：Settings System
  - 右：Multi Window

## 截图规范
- 统一使用深色主题
- 优先使用 `bilibili` 或 `obs` 主题
- 分辨率建议：
  - 横图：`1600 x 900`
  - 局部图：`1280 x 800`
- 保持窗口阴影、标题栏、原生桌面边缘，增强桌面客户端可信度
